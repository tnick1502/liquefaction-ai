# Кастомные нейросети: подробное описание методов

> **Только методы, без результатов.** Здесь — архитектуры, уравнения, лоссы и инварианты трёх
> собственных моделей и общей физической/дифференцируемой машинерии. Числовые результаты — в `results/`
> после чистого прогона. Связано: [[../formulations/methods-paragraphs]] · [[../protocol/metrics]] ·
> [[../protocol/evaluation-protocol]].

Прогноз строится из **причинного префикса** наблюдённого порового давления PPR(N) до landmark-цикла
N₀ и статических свойств грунта. Все три модели предсказывают: (1) траекторию огибающей PPR(N) на
априорной сетке циклов `1..H` (H=`max_cycle_reference`=3000), (2) число циклов до разжижения
`N_liq`, (3) бинарный риск разжижения by-H, (4) гетероскедастичную неопределённость траектории.

Событие разжижения определено **единым порогом** `ru = PPR ≥ 0.95` (`LIQ_THRESHOLD`) — первый цикл
устойчивого пересечения на сырых поцикловых пиках.

## 1. Общая машинерия (разделяется тремя моделями)

### 1.1 Кодировщик контекста и латент θ
Статические признаки, сводка префикса и сам причинный префикс (obs+mask) кодируются остаточным MLP
(`ResidualMLP`) в скрытое представление `encoded`. Из него головы `μ`, `logvar` задают латент θ.
θ интерпретируется как **физически ограниченные** параметры (sigmoid-проекции в допустимые диапазоны:
скорости накопления, экспоненты, пороги z0, крутизна κ и т.д.), а не произвольный embedding.

### 1.2 Физическая CRR(N)
Циклическое сопротивление разжижению CRR считается из свойств грунта (`physics/crr_physical`):
базовая кривая по относительной плотности/грансоставу с поправками на Ip, K0, затухание, частоту/режим.
Признаки обогащаются на стадии формирования данных; отсутствующие прямые измерения доопределяются
формулами (**K0 = 1 − sin φ** по Джекки; **Vs = √(G/ρ)** из small-strain G0), а provenance
(прямое измерение vs формула/fallback) сохраняется индикаторами `miss_K0`/`miss_vs`/… — они входят в
статические признаки, но НЕ являются пропусками в датасете.

### 1.3 Дифференцируемые примитивы (`training/losses.py`)
- **`interpolated_crossing(r, cycles, thr)`** — N_liq как линейно-интерполированный момент пересечения
  порога монотонной огибающей PPR. Straight-through: forward = точное пересечение на кусочно-линейной
  кривой (лежит ТОЧНО на кривой, не смещено размазыванием по лог-сетке), backward = гладкая CDF первого
  достижения `σ(β(r−thr))` (градиент есть и до фактического пересечения). Нет пересечения → цензура на
  последнем цикле сетки.
- **`monotone_clip(r, hi=1.0)`** — проекция PPR на монотонно неубывающую и физически ограниченную
  `ru∈[0,1]` траекторию (`cummax` + clamp). Модельное допущение недренированного монотонного накопления.
- **`monotone_residual_scale(r, res, span=0.1)`** — обучаемая коррекция траектории, **сохраняющая
  монотонность**: масштабирует неотрицательные приращения множителем `1+0.1·tanh(res)∈[0.9,1.1]`. Кривая
  остаётся неубывающей при любом residual → модель не может стать «physically unreliable».
- **`masked_censored_nliq_loss`** — цензур-корректный лосс N_liq: для разжижившихся (точный event-time)
  абсолютная ошибка, для наблюдаемых non-liq — **односторонний** штраф только за занижение (предсказание
  раньше точки цензуры). Незавершённые (исход by-H неизвестен) исключены маской.
- **`crossing_margin_loss`** (stable-only no-cross penalty) — штраф за ложное пересечение порога у
  **завершённых** non-liq (`regime_stable`); liquefied и незавершённые не штрафуются.
- **`gaussian_mixture_nll` / `energy_crps`** — proper scoring для MC-микстуры по θ∼flow (калибровка
  разброса потока под предиктивную ошибку).

### 1.4 Curve-first N_liq (единообразно во всех трёх моделях)
Публикуемый `N_liq` = `interpolated_crossing` собственной средней PPR-кривой модели → значение лежит
на кривой по построению (coherence-gap = 0; метрика `N_liq_Curve_Coherence_MAE`). Отдельная
**direct head** (`nliq_head`) остаётся **auxiliary**-предиктором (робастнее на OOD), обучается
собственным censored-лоссом и `consistency`-регуляризатором, тянущим её к пересечению кривой там, где
событие реально наблюдалось. Переключатель `report_nliq_from_curve` (по умолчанию True).

### 1.5 Физические инварианты выхода (все модели)
- PPR(N) монотонно неубывающая, `ru ≤ 1`;
- урон z(N) необратим (монотонно неубывающий);
- триггер события g(N) поглощающий (монотонно неубывающий) — `cummax` + проекция z гарантируют, что
  «разжиженный» образец не «разразжижается» обратно;
- `Physics_Violation_Rate = 0` по построению (проекция, не штраф) — тест `test_physics_state_invariants`.

## 2. DPI-Flow (`models/dpi_flow.py`, класс `DPIFlow`) — headline-модель

**Идея.** Амортизированный вывод физически-ограниченных θ через **conditional RealNVP** + прогон θ
через **аналитический дифференцируемый ODE-слой** разжижения.

**Поток θ.** `ConditionalCouplingFlow` — настоящий conditional RealNVP: latent-зависимые coupling-слои
со `scale`/`translate`, корректная density-objective с log-det Jacobian (а не affine-смещение контекста).
Разброс θ — это **conditional (амортизированный) posterior по входу**, НЕ байесовская эпистемическая
неопределённость по весам. Headline использует нормированную плотность (`calibration_steps=0`);
test-time градиентная доводка θ по префиксу — только абляция (не имеет заявленной плотности потока).

**ODE-слой (`AnalyticalLiquefactionLayer`).** Явная схема по сетке циклов интегрирует связанные
уравнения повреждения `dz` и порового давления `dr` под управлением отношения CSR/CRR, с до-/пост-
событийными ветвями, смешиваемыми мягким триггером `g`. Клампы `z∈[0,0.999]`, `r∈[0,1]`. Поверх —
`monotone_residual_scale` (малый zero-init резидуал) и `monotone_clip(hi=1.0)`.

**N_liq.** `soft_first_hitting` (обёртка над `interpolated_crossing`) по средней PPR-кривой; g не
используется в оценке момента (давал ложные ранние срабатывания).

**Головы.** Дискриминативный риск (`risk_clf` из контекста) + физический prior через обучаемый гейт
`prior_gate` + `RiskHead`; гетероскедастичная `SeqLogvarHead`; fold-local variance-scaling `calib_log_scale`.

**Лосс.** `L = L_traj(gaussian_nll | MC mixture-NLL + energy-CRPS) + 0.8·L_risk(BCE, observed) +
0.3·L_rank(soft-AUC) + 0.25·L_censored_Nliq + 0.03·monotonicity + 0.03·boundedness + 0.01·smoothness +
0.02·KL(flow) + crossing_margin_weight·L_margin + L_observed_aux`.

**Предиктив.** MC-прогон θ∼flow: `traj_var = aleatoric + epistemic`, headline N_liq пересчитывается на
MC-средней кривой (чтобы точечная оценка лежала на публикуемой траектории).

## 3. DPI-EVT (`models/dpi_evt.py`, класс `DPIEvtNet`, наследник `EVTNeuralSSM`)

**Идея.** Рекуррентный **EVT-движок** (`_engine`), интегрирующий структурированное состояние
`h=[z,r,c,g]` под физикой CRR/CSR, с опциональным conditional flow над θ и curve-first N_liq.

**Движок.** На каждом цикле: отношение `CSR/CRR_dyn` → `φ=softplus`, приращения `dz/dr` из физических
членов (степенные по `(1−z)`,`(1−r)`, фоновая densification `β/(N+τ)`, диссипация `−γ·r`) плюс малая
нейронная поправка, действующая в основном через вспомогательное состояние `c` (сохраняя физическую
доминанту). Интегратор Хойна (RK2). **Урон z спроецирован на неубывание** (`z=max(z,z_prev)`), `r∈[0,1]`,
триггер `g=σ(κ(z−z0))` монотонен по построению + `cummax` на выходе.

**CRR-пара.** `_compute_crr_pair` возвращает динамическую/выходную/consistency-версии CRR
(регуляризатор монотонности CRR по циклам).

**N_liq.** curve-first (`report_nliq_from_curve=True`): `interpolated_crossing` кривой; `pdf/mass`
(мягкая масса пересечения) сохраняются для supervision CRR-в-момент-разжижения. Direct head —
auxiliary, обучается censored-лоссом (`nliq_head_aux_weight`) + `nliq_consistency`.

**Специфические регуляризаторы.**
- `joint`-consistency единого состояния: `CRR(N_liq)≈CSR_app`, `damage(N_liq)≈0.90`;
- `overshoot` = односторонний штраф `relu(PPR_pred − PPR_obs)` у non-liq (потолок — **наблюдаемая**
  кривая, безопасно для правоцензурированных); адресует коллапс режима «нет разжижения»;
- `trigger_noliq` = подавление триггера на `regime_stable`;
- `switch_reg`, `state_smooth`, `boundedness`.

**Flow.** `use_flow` (по умолчанию может быть выключен для устойчивости на длинном обучении) — тот же
`ConditionalCouplingFlow`. Предиктив аналогичен DPI-Flow (curve-first точечная оценка на MC-средней).

## 4. EVT-NeuralSSM (`models/evt_ssm.py`, класс `EVTNeuralSSM`) — базовая структурная модель

**Идея.** Событийно-переключаемая нейронная **state-space** модель без conditional-flow над θ:
эволюция `h=[z,r,c]` с мягким переключением до/после события `g=σ(κ(z−z0))`, нейронная поправка только
через `c`. Интегратор Хойна. Инварианты те же (монотонные z/g, `r≤1`).

Это более «лёгкая» физическая модель (контраст к DPI-Flow/DPI-EVT: без амортизированного потока θ).
После унификации она тоже **curve-first**: публикует `interpolated_crossing` собственной кривой
(coherence-gap = 0), direct head — auxiliary. Лосс: traj + risk + rank + censored N_liq (кривая) +
head-aux + nliq_consistency + switch/smooth/boundedness + joint + overshoot + trigger_noliq + KL +
observed_aux.

## 5. Позиционирование трёх моделей
Все три — **physics-structured** предсказатели с одинаковыми физическими инвариантами и единым
curve-first протоколом N_liq. Различие — в способе вывода θ и глубине структуры:
- **DPI-Flow** — амортизированный conditional-flow posterior + аналитический ODE-слой (headline);
- **DPI-EVT** — рекуррентный EVT-движок (+ опц. flow), богатая пост-событийная физика и CRR-пара;
- **EVT-NeuralSSM** — базовая event-switched SSM без flow (нижняя граница «сколько даёт структура»).

Внешние бейзлайны (Transformer, GRU, TCN, NSF, CatBoost, PINN) и протокол оценки — см.
[[../protocol/baselines]], [[../protocol/evaluation-protocol]], [[../protocol/metrics]].
