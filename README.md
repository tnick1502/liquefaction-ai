# liquefaction-ai

Моделирование разжижения грунтов при циклическом нагружении с помощью
физически-структурированных нейросетевых моделей **DPI-Flow** и **EVT-NeuralSSM**.
Проект строит воспроизводимый исследовательский пайплайн: от генерации физически
согласованных синтетических данных до анализа метрик уровня A* с интерактивной
визуализацией.

## Структура проекта

```
src/liquefaction_ai/        # пакет (вся логика и модели)
  config.py, constants.py   # конфигурация эксперимента и таксономия грунтов/нагружений
  data/                     # генерация популяции, грансостав, профиль грунта, сплиты, I/O
    grainsize.py            #   11 фракций, классификация PLAXIS и ГОСТ type_ground
    soil_profile.py         #   полный геотех. профиль грунта (физика + механика + CRR-параметры)
    synthetic.py            #   сборка популяции и интегрирование ODE повреждения/PPR/триггера
  physics/                  # аналитические модели CRR/PPR + физическая модель CRR(N)
    crr_physical.py         #   CRR(N)=β/N^(1−α) из физсвойств (перенос model.py)
  models/                   # нейросети: базовые (MLP/GRU/TCN) и DPI-Flow / EVT-NeuralSSM
  training/                 # функции потерь, цикл обучения, grid search, сохранение моделей
  evaluation/               # метрики качества, агрегаты, эксперименты
  viz/                      # Plotly-тема, построители графиков (с флагом save), кривые CRR/PPR

notebooks/
  1_data_preparation/       # 1.1 генерация → data/, 1.2 анализ, 1.3 анализ CRR-параметров, 1.4 сплиты
  2_model_training/         # 2.1 базовые, 2.2 DPI-Flow, 2.3 EVT-NeuralSSM (grid search → models/)
  3_evaluations/            # 3.1 метрики, 3.2 абляции/OOD, 3.3 разбор кейсов
  crr_models.ipynb          # справочные графики аналитических моделей CRR
  ppr_models.ipynb          # справочные графики аналитических моделей PPR

data/    demo_run/          # артефакт популяции (генерируется ноутбуком 1.1)
models/  <модель>/          # обученные модели с гиперпараметрами и историей
results/ figs/              # сохранённые фигуры (.html + .png)
```

## Пайплайн

1. **Подготовка данных (папка 1).** Генерируется синтетическая популяция грунтов с полным
   геотехническим набором: гранулометрический состав (11 фракций), физические свойства
   (плотности, влажность, пределы Аттерберга, пористость, органика, карбонатность),
   классификации по ГОСТ (`type_ground`, 9 типов) и по PLAXIS, механические параметры
   (E, c, φ) и все входы физической модели CRR. Кривая сопротивления строится физически:
   **CRR(N) = β / N^(1 − α)**, где α и β выводятся из свойств грунта (model.py). Далее —
   история CSR(N), интегрирование ODE повреждения `z(N)`, порового давления `PPR(N)` и
   триггера `g(N)`, разведочный анализ, **глубокий анализ параметров CRR** и стратифицированное
   разбиение train/val/test.
2. **Обучение моделей (папка 2).** Обучаются три базовые модели (MLP, GRU, TCN) и две
   физически-структурированные (DPI-Flow, EVT-NeuralSSM); каждая сохраняется в
   `models/<имя>/` с гиперпараметрами, весами и историей обучения.
3. **Оценка (папка 3).** Считается полный набор метрик (траекторные MSE/MAE/RMSE,
   AUROC/AUPRC/Brier/ECE, калибровка, покрытие интервалов), проводятся абляции,
   out-of-distribution тесты и качественный разбор кейсов.

## Визуализация и сохранение фигур

Все графики строятся на Plotly в едином стиле (`liquefaction_ai.viz`). Построители
принимают флаг `save`: при `save=True` фигура сохраняется в `results/figs/` под именем
вида `{номер_папки}_{подноутбук}_{название}` (например `2_1_training_curves`) — в
интерактивном `.html` и статическом `.png`. В ноутбуках сохранением управляет переменная
`SAVE_FIGS`.

## Запуск

```bash
poetry install
poetry run jupyter lab
```

Ноутбуки выполняются по порядку: папка 1 → папка 2 → папка 3. Каждый ноутбук
самодостаточен: он сам находит корень репозитория и добавляет `src` в путь импорта.
